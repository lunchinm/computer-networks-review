6.1 应用层概述

本章核心问题：网络应用程序如何通过应用层协议实现具体的网络服务？

应用层协议的特点 重点

  • 运行在端系统中，使用运输层提供的服务
  • 每个应用层协议都是为了解决某一类应用问题
  • 协议定义了：报文类型、语法、语义、时序规则

6.2 DNS 域名系统 必考

DNS 的作用

DNS (Domain Name System)——将主机名（域名）转换为 IP 地址的分布式数据库系统。

为什么需要 DNS？人记域名容易，计算机需要 IP 地址来通信。DNS 是互联网的核心基础设施。

域名结构 重点

域名是分层的树形结构：

  • 根域名服务器（13 组，a.root-servers.net ~ m.root-servers.net）
  • 顶级域名 TLD：com, org, net, edu, cn, jp 等
  • 二级域名：如 baidu.com, google.com
  • 三级/子域名：如 www.baidu.com, mail.google.com

DNS 查询过程 重点

  1. 主机向本地域名服务器发出 DNS 查询（递归查询）
  2. 本地域名服务器向根域名服务器查询（迭代查询）→ 获得 TLD 服务器地址
  3. 向 TLD 服务器查询 → 获得权威域名服务器地址
  4. 向权威域名服务器查询 → 获得最终的 IP 地址
  5. 本地域名服务器缓存结果并返回给主机

DNS 使用 UDP 端口 53（大多数查询），长响应或区域传送则用 TCP。

6.3 万维网 WWW 必考

HTTP 协议 必考

HTTP (HyperText Transfer Protocol)——Web 上的客户—服务器程序之间的通信协议。

版本特点
HTTP/1.0非持续连接，每次请求/响应后断开 TCP
HTTP/1.1持续连接（keep-alive），支持流水线
HTTP/2.0二进制分帧、多路复用、头部压缩、服务器推送

HTTP 请求方法 重点

方法含义安全幂等
GET获取资源是是
POST提交数据否否
PUT更新/上传资源否是
DELETE删除资源否是
HEAD获取首部（不返回主体）是是

HTTP 状态码 高频

范围含义常见
1xx信息100 Continue
2xx成功200 OK, 201 Created, 204 No Content
3xx重定向301 永久重定向, 302 临时重定向, 304 Not Modified
4xx客户端错误400 Bad Request, 403 Forbidden, 404 Not Found
5xx服务器错误500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable

非持续连接 vs 持续连接 必考

类型HTTP版本方式开销
非持续连接HTTP/1.0每个对象需要单独一个 TCP 连接大（多次握手+慢开始）
持续连接HTTP/1.1+一个 TCP 连接传输多个对象小（一次握手即可）

为什么 HTTP/1.1 使用持续连接？网页包含多个对象（HTML+图片+CSS+JS），非持续连接需要大量 TCP 连接/释放，开销巨大。

Web 缓存/代理服务器 重点

Web 缓存——存储最近请求过的Web对象副本。用户请求时，先检查代理服务器是否有缓存副本且未过期，如果有则直接返回。

优点：减少响应时间、减少网络流量、减轻服务器负载。

6.4 电子邮件 重点

电子邮件系统的三个主要协议：

协议端口作用特点
SMTP25发送邮件（推送）TCP，ASCII 文本
POP3110收取邮件（拉取）下载到本地，服务器删除
IMAP143收取邮件（拉取）在服务器管理，不下载

6.5 文件传输 FTP 高频

FTP——使用 TCP 的可靠文件传输协议。需要建立两个 TCP 连接：

  • 控制连接：端口 21，传送控制命令
  • 数据连接：端口 20，传送文件数据
高频

6.6 DHCP 动态主机配置协议 重点

DHCP——自动给主机分配 IP 地址、子网掩码、默认网关、DNS 服务器等配置信息。

DHCP 基于 UDP，使用端口 67（服务器）和 68（客户端）。

过程：DHCP 发现 → DHCP 提供 → DHCP 请求 → DHCP 确认（DORA）。

协议运输层端口用途
DNSUDP/TCP53域名解析

SMTPTCP25发送邮件
POP3TCP110收取邮件
